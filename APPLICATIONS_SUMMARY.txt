================================================================================
                        KEY ADAPTATION POINTS
================================================================================

1. CRITERIA TYPES
   [√] Beneficial (Maximize)    : Quality, Experience, Capacity
   [√] Non-Beneficial (Minimize): Cost, Distance, Time, Risk
   [√] Binary                   : Certification, Compatibility

2. ASSIGNMENT TYPES
   [√] 1-to-1 (Hungarian)       : Employee-Project, Patient-Doctor
   [√] 1-to-Many (Greedy)       : Teacher-Students, Server-Jobs
   [√] Many-to-Many             : Requires custom algorithm

3. WEIGHTING STRATEGIES
   [√] Uniform                  : All criteria equal
   [√] Requirement-based        : Weight by importance
   [√] Custom                   : Domain-specific weights
   [√] Learning-based           : ML-optimized (future work)

4. CONSTRAINT TYPES
   [√] Capacity                 : Maximum workload per resource
   [√] Compatibility            : Must-have requirements
   [√] Temporal                 : Availability windows
   [√] Geographic               : Distance/location limits

================================================================================
                        REAL-WORLD EXAMPLES
================================================================================

EXAMPLE 1: Hospital Emergency Room
──────────────────────────────────
Profiles:    Doctors (10 specialists)
Activities:  Patients (15 urgent cases)
Skills:      Medical specializations (cardiology, trauma, etc.)
Method:      Greedy (more patients than doctors)
Criteria:    Specialization match (max), Wait time (min)
Result:      Optimal patient-doctor assignments in <1 second

EXAMPLE 2: IT Project Staffing
───────────────────────────────
Profiles:    Developers (20 available)
Activities:  Projects (20 active projects)
Skills:      Programming languages, frameworks, domain knowledge
Method:      Hungarian (equal dimensions)
Criteria:    Technical match (max), Experience (max)
Result:      Optimal 1-to-1 developer-project pairing

EXAMPLE 3: Supply Chain Optimization
─────────────────────────────────────
Profiles:    Suppliers (50 vendors)
Activities:  Purchase Orders (100 orders)
Skills:      Product categories, quality ratings, delivery capability
Method:      Greedy (more orders than suppliers)
Criteria:    Quality (max), Cost (min), Delivery time (min)
Result:      Cost-optimized supplier selection
